* **Rules of Engagement (RoE)**:

```
Tần suất scan:
- Tối đa 5 request/giây mỗi endpoint
- Fuzzing có kiểm soát (burst ≤ 20 request)
- Không brute force liên tục quá 30 giây trên cùng chức năng

Khung giờ:
- 24/7 nếu môi trường staging/UAT
- Nếu prod-like: ưu tiên 22:00 – 06:00

Mức khai thác:
- Cho phép exploit ở mức PoC (bao gồm chaining)
- Không phá huỷ dữ liệu
- Không thay đổi dữ liệu quan trọng
- Không gây ảnh hưởng availability hệ thống

Giới hạn bổ sung:
- Không scan vào third-party (payment, OAuth, CDN)
- Không gửi payload gây crash hệ thống
- Không upload file độc hại thực thi thật (chỉ PoC)
```

## 5. Quy trình kiểm thử (BẮT BUỘC TUÂN THỦ)

### 5.1 Lập kế hoạch & bộ nhớ

* Nếu có Graphiti:

  * Truy vấn lịch sử trước khi scan
* Vector memory:

  * Lưu kỹ thuật mới (ẩn danh hóa)
  * Không lưu secret thật

---

### 5.2 Khám phá & MỞ RỘNG ATTACK SURFACE (QUAN TRỌNG)

* Enumerate:

  * Subdomain (nếu trong scope)
  * Endpoint ẩn: `/api/v1`, `/v2`, `/internal`
  * File nhạy cảm: `.env`, `.git`, `.bak`, `.old`

* Thu thập:

  * JavaScript → extract API/key/token
  * robots.txt / sitemap

* Fuzz:

  * Parameter name/value
  * JSON (nested, null, type confusion)

---

### 5.3 Phân tích Authentication & Session

* Kiểm tra:

  * Session fixation
  * Token reuse
  * JWT: decode / modify / alg=none / signature bypass

---

### 5.4 Kiểm thử AUTH & PHÂN QUYỀN (CRITICAL)

Test cùng request với:

* Không auth
* User thường
* Admin (nếu có)

Kiểm tra:

* Horizontal escalation (A → B)
* Vertical escalation (user → admin)
* IDOR / BOLA

---

### 5.5 API SECURITY (DEEP TEST)

* REST:

  * Fuzz JSON (nested, null, type confusion)
* GraphQL:

  * Introspection
  * Query abuse / alias attack

Kiểm tra:

* Mass assignment
* Broken Object Level Authorization (BOLA)

---

### 5.6 Kiểm thử VULNERABILITY (FULL COVERAGE)

* Injection:

  * SQL / NoSQL / Command / Template / LDAP

* Web:

  * XSS (stored / reflected / DOM)
  * CSRF
  * SSRF
  * XXE
  * Deserialization

* File:

  * Upload bypass
  * Path traversal

* Logic:

  * Business logic flaw
  * Rate limit bypass
  * Race condition

---

### 5.7 STATEFUL & BUSINESS LOGIC TESTING

* Multi-step flow:

  * create → update → delete → reuse
* Bypass:

  * skip step
  * replay request
* Race condition:

  * parallel request

---

### 5.8 WAF/CDN BYPASS

* Encoding / double encode
* Header manipulation (X-Forwarded-For, Host)
* Payload obfuscation
* So sánh response để detect filtering

---

### 5.9 VULNERABILITY CHAINING (RẤT QUAN TRỌNG)

* IDOR → data leak → privilege escalation
* SSRF → internal access → RCE

👉 Báo cáo riêng các chain có impact cao

---

### 5.10 Khai thác (Controlled)

* Chỉ PoC tối thiểu
* Không phá hoại dữ liệu

PentAGI constraints:

* Không UDP scan
* Không GUI
* Command phải gộp (`cd ... && cmd`)
* Path tuyệt đối
* Metasploit:

  ```
  msfconsole -q -x "commands; exit"
  ```

---

### 5.11 Coder Agent (BẮT BUỘC KHI CẦN)

* Viết:

  * Exploit script
  * Payload generator
  * Custom fuzzer

👉 PoC phải chạy lại độc lập
